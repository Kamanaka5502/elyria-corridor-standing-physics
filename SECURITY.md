This repository is a protected public proof surface. It is not a production deployment package and must not be treated as a complete corridor-standing enforcement system.

## Supported public review scope

Security review is limited to the public demonstration and documentation files in this repository:

```text
README.md
CORRIDOR_STANDING_LAW.md
SOURCE_ANCHORS.md
SCENARIO_INDEX.md
PROTECTED_SCOPE.md
LICENSE.md
NOTICE.md
assets/corridor_standing_front_surface.svg
```

## Non-disclosure boundary

Do not submit, request, or publish:

```text
private keys, tokens, secrets, or credentials
real financial settlement data
real regulated data-transfer payloads
real government, energy, clinical, aerospace, or infrastructure data
production endpoint details
private law bundles
protected admissibility compiler logic
customer-specific corridor policies or evidence envelopes
NDA-bound formal proofs
protected substrate architecture
extreme-field substrate internals
```

## Public issue posture

Public issues should be limited to:

```text
documentation ambiguity
broken public links
unsafe interpretation risk in public wording
license / attribution clarity
public-scope boundary clarity
```

Do not publish sensitive operational details in issues or pull requests.

## Intended public behavior

Corridor Standing Physics must remain public-safe:

```text
valid endpoints do not prove valid transfer
transfer relation standing must be resolved before effect binds
closure standing is part of admissibility
unsafe or incomplete corridors fail closed
```

## Public proof limitation

The public repository is intentionally simplified. It demonstrates the category and public-facing law without exposing protected internals.

The absence of production internals, extreme-field substrate, and private law logic is intentional.
